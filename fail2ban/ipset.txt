1. 现在：装 ipset + 手动黑名单（约 10 分钟，一次性）                                                                                                     
                                                                                                                                                           
  1.1 装包
                                                                                                                                                           
  sudo apt update
  sudo apt install -y ipset iptables-persistent                                                                                                            
                  
  ▎ 装 iptables-persistent 时会弹窗问"是否保存当前 iptables 规则"——选 Yes 都行，反正下面会重新保存。                                                       
   
  1.2 创建 ipset 黑名单集合                                                                                                                                
                  
  sudo ipset create blacklist hash:net family inet hashsize 4096 maxelem 65536
                                                                                                                                                           
  参数说明：
  - hash:net：支持网段（/24）级别的封禁                                                                                                                    
  - maxelem 65536：最多 6.5 万条（够用很久）                                                                                                               
                                            
  1.3 加入需要封的 IP 段                                                                                                                                   
                                                                                                                                                           
  之前数据里 top 攻击源对应的网段：                                                                                                                        
                                                                                                                                                           
  sudo ipset add blacklist 130.12.181.0/24
  sudo ipset add blacklist 38.91.104.0/24
  sudo ipset add blacklist 77.110.127.0/24                                                                                                                 
  sudo ipset add blacklist 205.164.114.0/24
  sudo ipset add blacklist 111.174.126.0/24                                                                                                                
                                                                                                                                                           
  1.4 接到 iptables（让 ipset 真正起作用）                                                                                                                 
                                                                                                                                                           
  # -I INPUT 1 = 插到 INPUT 链最前面，比 fail2ban 的 f2b-sshd 还要早执行                                                                                   
  sudo iptables -I INPUT 1 -m set --match-set blacklist src -j DROP                                                                                        
                                                                                                                                                           
  1.5 验证                                                                                                                                                 
                                                                                                                                                           
  # 看集合内容    
  sudo ipset list blacklist | head -20
                                                                                                                                                           
  # 看 iptables 规则
  sudo iptables -nvL INPUT --line-numbers | head -5                                                                                                        
  # 第 1 行应该是: ... match-set blacklist src DROP                                                                                                        
                                                                                                                                                           
  # 等几分钟后看 pkts 数字（被拦截的包数）—— 应该一直涨                                                                                                    
  sudo iptables -nvL INPUT --line-numbers | head -2                                                                                                        
                                                                                                                                                           
  1.6 持久化（重启不丢）                                                                                                                                   
                                                                                                                                                           
  # 1) 保存 iptables 规则                                                                                                                                  
  sudo netfilter-persistent save

  # 2) 保存 ipset 内容                                                                                                                                     
  sudo mkdir -p /etc/iptables
  sudo ipset save > /etc/iptables/ipsets                                                                                                                   
                                                                                                                                                           
  # 3) 创建 systemd 服务，开机时先恢复 ipset，再恢复 iptables
  sudo tee /etc/systemd/system/ipset-restore.service > /dev/null <<'EOF'                                                                                   
  [Unit]                                                                                                                                                   
  Description=Restore ipset rules
  Before=netfilter-persistent.service                                                                                                                      
  DefaultDependencies=no
                                                                                                                                                           
  [Service]
  Type=oneshot                                                                                                                                             
  ExecStart=/sbin/ipset restore -exist -f /etc/iptables/ipsets
  RemainAfterExit=yes

  [Install]                                                                                                                                                
  WantedBy=multi-user.target
  EOF                                                                                                                                                      
                  
  sudo systemctl daemon-reload
  sudo systemctl enable ipset-restore.service
                                                                                                                                                           
  # 4) 测试一下重启逻辑（不真重启，只 dry-run 启动服务）
  sudo systemctl start ipset-restore.service                                                                                                               
  sudo systemctl status ipset-restore.service --no-pager
                                                                                                                                                           
  ▎ ⚠️  以后改了 ipset 内容（加 IP / 删 IP），都要重新 save：
  ▎ sudo ipset save > /etc/iptables/ipsets                                                                                                                 
  ▎ 否则重启后改动消失。

  2. 怎么查看封禁数量                                                                                                                                      
                     
  2.1 几个直接命令
                                                                                                                                                           
  # fail2ban 报告（最直观）
  sudo fail2ban-client status sshd | grep -E "Currently|Total"                                                                                             
  # 输出形如:                                                                                                                                              
  # |- Currently failed:    0
  # `- Total failed:        53841992                                                                                                                       
  # |- Currently banned:    6           ← 这个是关键数字
  # `- Total banned:        18211                                                                                                                          
                                                                                                                                                           
  # 直接数 iptables 规则数（更精准）                                                                                                                       
  sudo iptables -nvL f2b-sshd 2>/dev/null | tail -n +3 | wc -l                                                                                             
                                                                                                                                                           
  # 数 INPUT 链全部规则数
  sudo iptables -nvL INPUT 2>/dev/null | tail -n +3 | wc -l                                                                                                
                                                                                                                                                           
  关键看的是 Currently banned（同时在封状态的 IP 数）= f2b-sshd 链当前规则数。Total banned 是历史总数，含已经过期解封的，不影响性能。                      
                                                                                                                                                           
  2.2 阈值参考                                                                                                                                             
                  
  ┌──────────────────┬──────────────────────────────┬────────────┐                                                                                         
  │ Currently banned │             影响             │    建议    │
  ├──────────────────┼──────────────────────────────┼────────────┤                                                                                         
  │ < 1,000          │ 无感                         │ 不用动     │
  ├──────────────────┼──────────────────────────────┼────────────┤                                                                                         
  │ 1,000 – 5,000    │ 几乎无感                     │ 不用动     │
  ├──────────────────┼──────────────────────────────┼────────────┤                                                                                         
  │ 5,000 – 10,000   │ 高 PPS 时可能略有 CPU 开销   │ 可以提前切 │
  ├──────────────────┼──────────────────────────────┼────────────┤                                                                                         
  │ > 10,000         │ 每个进出包都线性扫规则，明显 │ 该切了     │
  └──────────────────┴──────────────────────────────┴────────────┘                                                                                         
                  
  ▎ 真正的判断标准其实是"系统 load 是否上升 / SSH 是否变慢"。10k 是经验数字，不是硬阈值。                                                                  
                  
  2.3（可选）每天自动看一下                                                                                                                                
                  
  如果懒得手动查，加个 cron 简单监控：                                                                                                                     
                  
  sudo tee /etc/cron.daily/fail2ban-banlist-size > /dev/null <<'EOF'
  #!/bin/bash                                                                                                                                              
  COUNT=$(fail2ban-client status sshd 2>/dev/null | awk '/Currently banned/ {print $NF}')
  THRESHOLD=8000                                                                                                                                           
  if [ "$COUNT" -gt "$THRESHOLD" ]; then                                                                                                                   
      logger -t f2b-monitor "fail2ban currently banned=$COUNT exceeds $THRESHOLD, consider switching to ipset banaction"                                   
  fi                                                                                                                                                       
  EOF             
  sudo chmod +x /etc/cron.daily/fail2ban-banlist-size                                                                                                      
                  
  到时候 journalctl -t f2b-monitor 会有提醒。

  3. 将来切换到 ipset 版 fail2ban（封禁多了再做）
                                                                                                                                                           
  3.1 切换时机    
                                                                                                                                                           
  Currently banned 长期稳定在 8000+，或服务器 CPU 在网络高峰期变高，就切。                                                                                 
                                                                                                                                                           
  3.2 切换步骤（约 2 分钟）                                                                                                                                
                  
  # 1) 备份现状（万一要回滚）                                                                                                                              
  sudo iptables-save > /tmp/iptables.backup.$(date +%F)
  sudo ipset save > /tmp/ipsets.backup.$(date +%F)
                                                                                                                                                           
  # 2) 改一行 jail.local
  sudo sed -i 's/^banaction = iptables-allports$/banaction = iptables-ipset-proto6-allports/' /etc/fail2ban/jail.local                                     
                                                                                                                                                           
  # 3) 验证改对了
  grep "^banaction" /etc/fail2ban/jail.local                                                                                                               
  # 应该输出: banaction = iptables-ipset-proto6-allports                                                                                                   
                                                                                                                                                           
  # 4) 重启 fail2ban（它会自动清掉旧的 N 千条 iptables 规则，改用 ipset）                                                                                  
  sudo systemctl restart fail2ban                                                                                                                          
                                                                                                                                                           
  3.3 验证切换成功                                                                                                                                         
                                                                                                                                                           
  # 应该多了一个 fail2ban 自己管理的 ipset（名字一般是 f2b-sshd）                                                                                          
  sudo ipset list -n                                                                                                                                       
  # 输出形如:                                                                                                                                              
  # blacklist        ← 你手动建的                                                                                                                          
  # f2b-sshd         ← fail2ban 新建的                                                                                                                     
                                                                                                                                                           
  # f2b-sshd 链现在只有 1 条规则（通过 ipset 匹配），不再是 N 千条                                                                                         
  sudo iptables -nvL f2b-sshd                                                                                                                              
                                                                                                                                                           
  # fail2ban 状态正常
  sudo fail2ban-client status sshd                                                                                                                         
                                                                                                                                                           
  切换前后对比：
                                                                                                                                                           
  切换前:                                切换后:
  f2b-sshd 链:                          f2b-sshd 链:
    REJECT 1.2.3.4                        DROP match-set f2b-sshd src                                                                                      
    REJECT 2.3.4.5                      f2b-sshd ipset 内:                                                                                                 
    REJECT 3.4.5.6                        1.2.3.4                                                                                                          
    ... (8000 条)                         2.3.4.5                                                                                                          
                                          3.4.5.6                                                                                                          
                                          ... (8000 条)                                                                                                    
                  
  每个包扫 8000 条规则                   每个包查 1 个 hash → O(1)                                                                                         
                  
  3.4 持久化新状态                                                                                                                                         
                  
  sudo ipset save > /etc/iptables/ipsets
  sudo netfilter-persistent save                                                                                                                           
   
  ---                                                                                                                                                      
  一句话总结      
                                                                                                                                                           
  - 现在做：装 ipset + 手动黑名单 + 持久化（上面第 1 步全部）。
  - 平时只需：偶尔 sudo fail2ban-client status sshd | grep "Currently banned" 看一眼。                                                                     
  - 数字到 8000+ 时：改一行 jail.local + 重启 fail2ban + 重新 save，2 分钟搞定。
